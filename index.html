Introduction

A Bling is an ordered collection of items of any type. This set defines functional operations to collect and manipulate the members.

You create a Bling by wrapping up or selecting other items, sets, or subsets: 

	$("button") // css expressions
	$("") // or html
	$([1, "foo", {a: 'b'}]) // or anything

What makes BlingJS special?

While similar to jQuery, there are a number of important differences. Mostly, a philosophical belief in the power of programming with sets, and the consequences of that in the implementation. But also, a few neat tools that when combined give you quite a bit of power.

For complete documentation of everything BlingJS can do, see the API Documentation (which is generated live from the current source).

The biggest philosophical difference is that BlingJS always tries to return a set, rather than first elements, unless specifically requested by first, or last. Even methods that return values (like width or height) will return a set of values (one for each node in the source set), and if there is only one, you access it using first.

One of my big frustrations with jQuery is having to do stuff like this: 

	jQuery("div").map(function() {
		return $(this).html()
	})

Using BlingJS, this is equivalent to: 

Bling("div").html()

And, in the opposite direction: 

	jQuery("div").html() 
	=== Bling("div").html().first()

Set Operations

In order to make this set-focused philosophy work, a few core tools are needed. We highlight only two here, but there are many more, please do browse the Full API.

Zip

The zip operation selects of list of properties from a set of elements. Think of zip as being similar to a SELECT query in SQL.

The expression: 

	$("div").zip("style")

Will select a list of StyleDeclaration objects, one for each div on the page.

You can also zip compound names in order to extract precise information from a set. 

	$("div").zip("style.setProperty")

Not only does the compound form allow for precise extraction, but the above example also demonstrates another useful feature of zip:

When zip selects a function, it is bound to the object it is selected from.

What this means is that when you .zip("style.setProperty") you get back a set of functions, and when you call these functions directly, it is as if you were calling: div.style.setProperty().

In other words, the this value inside each returned function, will be a reference to the original style from which setProperty was extracted.

The call operation exists in order to call all the functions in the set with the same arguments.

If you specify multiple columns, you get a list of simple objects with only the desired fields. 

	$("div").zip("id", "className")
	== $([{'id':..., 'className':... },...])
Zap

Naturally, zap is the companion to zip. The zap operation allows assignment in a functional syntax. 

	$("span").zap("innerText", "Hello World")

Like zip, it also supports compound properties. 

	$("span").zap("style.color", "red")

Or, you can give a bunch of properties to set all at once. 

	$("span").zap({
		"style.color": "red",
		"innerText": "Error"
	})

If you want to assign a different value to the same property on each item, you can do this. 

	$("span").take(3).zap("style.color",
		["red", "green", "blue"])

If you use zap this way, you must make sure you have enough values for how many elements you select.

Html Operations

While BlingJS has all of the basic HTML and DOM manipulating operations that jQuery has (like html, text, after, append, etc.), it also has two simple systems that work together to add quite a bit more power.

Templates

The first such system that is built into the core is a simple template system, and it works by replacing substrings that look like these: %(id)d, %(name)s, and %(dollars).2f

All templates start with DOM nodes.

Once you have the DOM nodes you want to copy, you should usually initialize them with template. 

	$("#line-template").template({
		title: "default title",
	})

Initializing the nodes with template will set default values for the fields, and will remove the nodes from the DOM so that their un-rendered forms don't interfere with the page.

The render operation creates new nodes from templates. 

	$("#line-template").render({
		title: "This is my line."
		body: "This is my line's body."
	})

The initialization with template is optional, otherwise you will have to manage the source nodes and defaults yourself.

Synthesizing nodes

One of the handiest features in Bling is the synth operation. This lets you fabricate small DOM trees, given a CSS expression.

In other words, given a CSS expression, $.synth(css) will return the smallest DOM tree that satisfies the CSS.

Example: 

	$.synth("div#foo.bar")
	== $('
')

The above example used the global $.synth(), for creating nodes that are initially un-attached to anything. If you use $(selector).synth(...), the new nodes will be appended to $(selector) before they are returned.

The synth operation supports only a few CSS selectors:
.class <dot>any number
#id <hash>once
[attr=val]any number
+ <plus>between nodes
"quote"after a node

It supports the non-standard quote operator "...", like in this example: 

	$.synth('div "Hello" p "World"')
	== $('
Hello
World
')

If the adjacency operator (+) is not clear, you can see what it does in this example: 

	$.synth('p b "Hello" + i "World"')
	== $('
HelloWorld
')

Even though this is only a small subset of CSS selectors, you can still construct an aribtrary DOM tree by combining the supported selectors.

Synthing templates

You can use synth and template to fabricate template nodes quickly in code. 

	var paragraph = 
		$.synth('p b "%(title)s" + i "%(body)s"')
			.template({
				title: "Default Title",
				body: "Default Body"
			})
	// ... sometime later ...
	$("body").append(paragraph.render({
		title: "This is paragraph."
		body: "This is my paragraph's body."
	}))
